Introdução à Virtualização

Técnica que permite a criação de uma máquina virtual para funcionar dentro do sistema de um PC. É utilizada tanto por usuários comuns como por profissionais de TI e permite, entre outras coisas, a execução de SOs completos ou uma simulação do comportamento do sistema para quem trabalha com desenvolvimento de software e testes de segurança.

Histórico e origem da virtualização

A ideia surgiu na metade de 1960, quando os computadores tinham velocidade de processamento, mas ineficiência no tempo de cálculo, devido ao gerenciamento de processos ser feito manualmente pelo operador.

O conceito se consolidou na década de 70, sendo importante para o desenvolvimento de novas arquiteturas e novas tecnologias. Em 1972 a IBM lançou um mainframe capaz de executar simultaneamente diferentes SOs sob a supervisão de um programa de controle (hypervisor).

O primeiro computador comercial inteiramente projetado para virtualização foi o sistema 370 da IBM que, com o SO CP/CMS, permitia executar múltiplas instâncias simultaneamente. Logo depois veio o IBM z/VM, que se aproveitava da virtualização via hardware de forma mais completa, onde todas duas interfaces de hardware eram virtualizadas.

Com o passar dos anos, a virtualização foi caindo no esquecimento devido a criação de novas aplicações client/servidor e ao declínio da plataforma mainframe que perdeu força frente a ascensão da plataforma x86. E com a ampla adoção dos SOs Windows e Linux em servidores na década de 1990, isso estabeleceu a arquitetura x86 como padrão da indústria.

Em 1999, a VMWare Inc. introduziu o conceito de virtualização na plataforma x86 como uma maneira mais eficiente para utilizar o equipamento desta plataforma. Eles aproveitaram os servidores x86 para fornecer uma estrutura computacional que possibilitasse o total aproveitamento dos recursos computacionais destes servidores

Então, a partir de 2005 os fabricantes de processadores como Intel e AMD deram mais atenção a necessidade de melhorar o suporte via hardware em seus produtos. A Intel com sua tecnologia Intel VT e a AMD com a AMD-V.

Algumas ferramentas de virtualização

As ferramentas mais comuns de virtualização são destinadas à virtualização de desktops, tais como o Microsoft Virtual PC e o Virtual Box.
Outras ferramentas conhecidas:

VMware

É o software de virtualização para a plataforma x86 mais conhecido, com uma implementação completa de interface ao sistema convidado.
É útil em diversas aplicações como:
  • Ambientes de desenvolvimento
  • Ambientes de suporte
  • Migração e Consolidação
  • Simulação de Instalações de Rede
  • Balanceamento de Carga

Xen

Patrocinado pela Universidade de Cambridge - Inglaterra, é uma das principais soluções de virtualização para Linux

Faz implementação entre o Hardware e Software para plataforma X86. Com suporte a múltiplos sistemas convidados simultaneamente, excelente desempenho, isolamento e seu baixo custo, o Xen é o que especialistas esperavam para melhorar a virtualização no Linux.

TIPOS DE VIRTUALIZAÇÃO

Todos os tipos de virtualização têm pelo menos uma coisa em comum: a abstração de algo físico em partes discretas via software.

Seguem os principais tipos de virtualização:

Virtualização de rede

Usa os recursos disponíveis em uma rede e divide a largura de banda em canais distintos, onde se pode proteger cada canal separadamente e atribuir os canais a dispositivos específicos em tempo real.

As metas principais da virtualização de rede são melhorar a velocidade, a disponibilidade e a segurança das redes, além de aliviar a carga associada ao gerenciamento de rede. Dá a opção gerenciar arquivos centralmente, adicionar ou reatribuir mídia de armazenamento e compartilhar ou realocar o espaço de armazenamento entre os servidores, conforme necessário.

Virtualização de armazenamento

Fonte Figura: Click Aqui.

Usa o software para encontrar espaço de armazenamento físico de vários dispositivos e agrupá-lo no que parece ser um único dispositivo de armazenamento que as máquinas podem usar.

De forma resumida, o software recebe solicitações de E/S de dispositivos físicos e VMs e, em seguida, envia as solicitações para o local do armazenamento físico que faz parte do pool de armazenamento geral, tornando possível o gerenciamento de todo o armazenamento através de um console central.

Os benefícios desse tipo de virtualização incluem o gerenciamento simplificado de tipos de armazenamento heterogêneos, custos de hardware reduzidos e maior desempenho e confiabilidade.

Virtualização de desktop

Fonte Figura: Click Aqui.

Têm como objetivo principal isolar um SO de desktop que os usuários utilizam para acessá-lo.

Com a virtualização baseada em host ou em infraestrutura de desktop virtual (VDI), os usuários se conectam a uma área de trabalho virtual em uma rede usando um protocolo de exibição remota.

Os clientes podem ser desktops tradicionais, thin clients ou até mesmo dispositivos móveis, como smartphones ou tablets. Um usuário se conecta a uma VM que executa o SO e os aplicativos de que precisa.

Quando vários usuários se conectam a uma área de trabalho compartilhada, como é o caso dos Serviços de Área de Trabalho Remota da Microsoft, ela é conhecida como virtualização de área de trabalho hospedada compartilhada.

Virtualização de aplicativos

Se a única coisa que os usuários devem acessar a partir dessa área de trabalho compartilhada é um único aplicativo, isso é chamado de virtualização de aplicativos.

Um aplicativo é executado separadamente do dispositivo que o acessa. A virtualização de aplicativos possibilita que os administradores de TI instalem e atualizem uma versão de um aplicativo, em vez de executar as mesmas tarefas de gerenciamento várias vezes.

A abordagem mais comum à virtualização de aplicativos é baseada em servidor, em que o aplicativo é executado em uma VM, no data center ou em outro serviço de hospedagem, e os usuários interagem com o aplicativo em um protocolo de exibição remota.

Na verdade, um aplicativo não pode ser executado independentemente de um SO, então, de certa forma, a virtualização de aplicativos é a de desktops. Geralmente, sua aplicação de uso determina qual termo está correto.

Conceitos básicos da virtualização

A virtualização consiste na emulação de ambientes isolados, permitindo que diferentes SOs rodem dentro de uma mesma máquina, aproveitando ao máximo a capacidade do hardware. Esse aproveitamento é maior, pois fornece a possibilidade de vários ambientes de execução independentes a diferentes usuários em um mesmo equipamento físico.

Pode diminuir o poder dos SOs que, muitas vezes, restringem o uso do hardware, quanto à utilização de software. Isso acontece porque softwares normalmente só rodam no SO para o qual foram projetados para rodar. Diferentes sistemas operando em uma mesma máquina aumentam a gama de softwares que podem ser utilizados sobre o mesmo hardware.

Vantagens

Melhor aproveitamento da infraestrutura existente: executando vários serviços em um servidor ou conjunto de máquinas, pode-se aproveitar a capacidade de processamento destes equipamentos o mais próximo possível de sua totalidade.

Uso de sistemas legados: desde que se destine a ele uma máquina virtual compatível com o seu ambiente.

Diversidade de plataformas: possibilitando realizar testes de desempenho de determinada aplicação em cada uma delas.

Fonte Figura: Click Aqui.

Gerenciamento centralizado: facilitando o monitoramento dos serviços em execução.

Segurança e confiabilidade: como cada máquina virtual funciona de maneira independente das outras, um problema que surgir em uma delas - como uma vulnerabilidade de segurança - não afetará as demais.
